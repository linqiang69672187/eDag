Private Call

There are two modes to initiate a private call:

  • Tel mode: full-duplex private call mode of the trunking communication system. In this mode, the two parties can receive or transmit voices simultaneously.

  • PTT mode: half-duplex private call mode of the trunking communication system. At the same time, only one party can transmit voices, and the other party can only receive the voices. Both parties need to hold down PTT button to preempt the voice transmission permission.

The private call parameters are as follows:

Terminal ID

The ID of the terminal used by the mobile user

Name

The name of the mobile user

Unit

To which unit the mobile user belongs

Number

The serial number of the mobile user

Status

Call access request

The mobile user is calling the dispatch console

Call is being processed

The dispatch console calls out, waiting for the server to respond

The subscriber is ringing

The server has responded, waiting for reply

Call is set up

The user has replied, the server assigns the line resources and establishes the connection

Obtain authorization

The connection has been established, the voice permission has been obtained

PTT released

The user has released the PTT button, and the voice permission has been released

Dispatcher is speaking

The dispatcher has obtained the voice permission and is speaking

The subscriber is speaking

The user has obtained the voice permission after the connection is established

The subscriber hangs up

The connection is disconnected

Answer mode

Can be divided into auto-answer mode and non-auto-answer mode

Call mode

Tel mode

Full-duplex private call: Voice transmission and reception are done in parallel

PTT mode

Half-duplex private call: At the same time, only one party can send the voice, and the other party can only receive the voice. Both parties need to press the PTT button to preempt the voice permission.

 

Note:

A dispatcher can make a private call to at most one mobile user or one dispatch console at any one time.
