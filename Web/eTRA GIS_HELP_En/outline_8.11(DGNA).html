DGNA

DGNA (Dynamic Group Number Assignment) is equivalent to creating a temporary talkgroup.

The configuration user can create the dynamic group (see "Dynamic Group Information Maintenance"), while the dispatch user can add members to the dynamic group, and regroup the members.

This chapter mainly describes:

  • Add Dynamic Group Member

  • Dynamic Group Maintenance

The steps of adding members to a dynamic group are as follows:

1) Click [Dispatch/Call panel] on the menu bar, and click "Dynamic reorganization”. The “Dynamic reorganization” dialog box pops up, shown in Figure 1.1.

Figure 1.1. "Dynamic reorganization" dialog box

2) Enter the terminal ID and click <Add>, or click <Select> to select the user, and then select the dynamic group name, shown in Figure 1.2.

Figure 1.2. Add the dynamic group members

3) Alternatively, click "" in the map toolbar and select a region of one or more mobile users. The user list pops up. Click <Dynamic reorganization>, and then set the parameters.

4) Click <OK>. The system will display the execution process and result, shown in Figure 1.3. You can also click <Cancel the execution> to end the process.

Figure 1.3. Process prompt box

The steps of dynamic group maintenance are as follows:

1) Click [Dispatch/Dynamic group maintenance] on the menu bar. The maintenance dialog box pops up, shown in Figure 2.1.

Figure 2.1. Dynamic group maintenance

2) Click the dynamic group name on the left, and it will list all members of this group. You can filter the users based on ISSI or status, shown in Figure 2.2.

Figure 2.2. Select the dynamic group members

3) Select the users. If they are not regrouped, you can click <Recombination> to regroup them; if they are already regrouped, you can click <Termination of reorganization> to cancel the DGNA. You can also click <Remove members> to remove them from the dynamic group.
