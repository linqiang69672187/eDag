Appendix

Currently, the types of terminals can be divided into Sepura and MOTO. For the normal use of eTRA GIS system, both of the two types need to be configured correctly. Their programming methods are different.

  • Sepura Configuration Considerations

  • MOTO Configuration Considerations

Sepura Configuration Considerations

1) If you have shortened dialing configured in the Network template, it is always best to program the GPS command ISSI (or any command address) with the full 8 digits or more longer. For example, the terminal ISSI is 5 digits, then you should set the GPS command ISSI (or any command address) with the full 5 digits, or fill 0 in front of the GPS command ISSI (or any command address) without enough digits.

2) In the All parameters template, the default report distance cannot be set to 0. If it is set to 0, the distance report function will be disabled. The recommended setting is 0.1(100M).

3) There are following handling precautions for Sepura GPS air interfaces:

  • Immediate location report function: the dispatch user ISSI is the GPS command ISSI, shown in Figure 1.

  • Modify the destination: the dispatch user ISSI is as above.

MOTO Configuration Considerations

There are following handling precautions for MOTO GPS air interfaces:

  • Immediate location report function: The dispatch user ISSI must meet the conditions in Figure 3. For example, the GPS Authorised ISSI Base is A, the GPS Authorised ISSI Range is B, and the dispatch user ISSI is XX. Then A ≤ XX ≤ B.

Figure 3. MOTO ISSI parameter explanation

  • Location reporting enable/disable function: the dispatch user ISSI is as above.

  • Modify the destination: the dispatch user ISSI is as above.

  • Modify the maximum reporting interval: the dispatch user ISSI is as above. Principles: input the time of Multiples 10s on gis web, else rounded up in accordance with 10s.

Figure 4. Interval parameter explanation for MOTO
