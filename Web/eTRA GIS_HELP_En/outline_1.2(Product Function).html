Product Function

The functions of eTRA GIS are as follows:

  • Basis information maintenance

  • Marshaling information maintenance

  • Map function

  • Application function

  • Dispatch function

  • Log function

Basis Information Maintenance

Users can add, delete, query or modify the following information:

  • Unit information

  • Terminal information

  • Mobile user information

  • Dispatcher information

  • Configuration user information

  • Dispatch console information

  • User type information

  • Base station information

Marshaling Information Maintenance

Users can add, delete, query or modify the following information:

  • Group information

  • Multicast group information

  • Patch group information

  • Multi-select group information

  • BS group information

  • Dynamic group information

Map Function

The map functions include:

  • Zoom in

  • Zoom out

  • Pan

  • Navigation bar

  • Region selection

  • Distance measurement

  • Switching

  • Hawkeye

Application Function

The application functions include:

  • Lock: Once in lock mode, the user will be marked with a special icon, and the map will make automatic adjustments constantly to make sure that the user is always in the visible region.

  • Rapid positioning : Get the target user’s latitude and longitude information.

  • Electronic fence: Zone the mobile users in a specific electronic fence. When a user leaves the electronic fence, the dispatch console will send an alarm to the user.

  • Real-time display: When the system receives the position information of a mobile user, it will display the user’s icon and corresponding information on the screen.

  • Real-time trajectory: Display the user’s trajectories in lines at real time.

  • Historical trajectory: Replay the trajectories of a user in a certain period.

Dispatch Function

The dispatch functions include:

  • Private call: Make half-duplex or full-duplex private calls to mobile users.

  • Group call: Initiate a call to a single group, or receive a call from a group.

  • Multicast group call: Initiate a call to a multicast group, or receive a call from a multicast group.

  • Patch group call: The dispatch console can combine multiple groups into one patch group and initiate a call to the group.

  • Multi-select group call: The dispatch console can combine multiple groups into one multi-select group and initiate a call to the group.

  • Base station call: Base station call is broadcast call, including single-BS call, multi-BS call and all-BS call.

  • SMS: Send SMS to single user/multiple users/group users, or receive SMS from them.

  • Discreet listening: The dispatch console can monitor the communications of a user without being noticed. When the user being tracked has a call, the dispatch console can monitor the call.

  • Ambience listening: Ambience listening can only be set up when the user is idle. Once the listening is set up, the monitored terminal will automatically open its microphone to collect the surrounding voices, and transmit the voice packet to the dispatch console.

  • Break-in: Forced release is set up based on discreet listening. When a monitored user is in a private call, the dispatch console can break in the call.

  • Forced release: Forced release is set up based on discreet listening. When a monitored user is in a private call, the dispatch console can disconnect the call.

  • Kill: The dispatch console can kill a radio in the network, and the killed radio cannot perform any operation.

  • Revive: Revive is a reverse action of kill. The killed radio will recover to the normal working status after receiving the recovery instruction issued by the dispatch console.

Log Function

The log functions include:

  • System log: Display the system-level log and API call information in the system log window in the lower left corner.

  • Call log: Display the call information in the call log window in the lower left corner.

  • SMS log: Display the SMS information in the SMS log window in the lower left corner.

  • Operation log: Display the operation information in the operation log window in the lower left corner.
