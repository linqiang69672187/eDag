Break-In

Break-in is set up based on discreet listening. When a monitored mobile user is in a call, the dispatcher can break in the call to preempt the voice transmission permission, and replace the other user to communicate with the monitored user.

  • Break in Full-Duplex Call

  • Break in Half-Duplex Call

For example, “180008” has been monitored, and is in a full-duplex call with “160001”. The steps are as follows:

1) Click <Break-in>. Then the dispatcher can replace “160001” to communicate with “180008”, shown in Figure 1. The dispatcher can input voices directly.

Figure 1. Break in the full-duplex call

2) “180008” hangs up the terminal or the dispatcher clicks <End call> to end the current call, shown in Figure 2.

Figure 2. End the full-duplex call

3) At this time, the discreet listening has not yet ended. The dispatcher can still monitor “180008” if another call generates.

4) Click <End>. The discreet listening ends.

For example, “180008” has been monitored, and is in a half-duplex call with “160001”. The steps are as follows:

1) Click <Break-in>. Then the dispatcher can replace “160001” to communicate with “180008”, shown in Figure 3.

Figure 3. Break in the half-duplex call

2) Press <PTT> to obtain the transmission permission, shown in Figure 4. After the dispatcher releases <PTT>, “180008” can hold down the PTT button to obtain the permission.

Figure 4. Dispatcher obtains permission

3) “180008” hangs up the terminal or the dispatcher clicks <End call> to end the current call.

4) At this time, the discreet listening has not yet ended. The dispatcher can still monitor “180008” if another call generates.

5) Click <End>. The discreet listening ends.

Note:

Break-in can only be realized in discreet listening status.
